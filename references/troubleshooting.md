# 故障排查指南

## 排查流程总览

```
读取失败/温度不刷新/数据异常
    │
    ▼
Step 1: 硬件检查
    ├─ 传感器是否在线（目视检查、测电阻）
    ├─ 天线连接是否牢固
    └─ 采集器是否上电（指示灯状态）
    │
    ▼
Step 2: 信号质量检查
    ├─ RSSI 是否在 -75dBm 以上
    └─ 读取距离是否在典型值范围内
    │
    ▼
Step 3: 协议检查
    ├─ 波特率/校验位是否匹配（9600/8N1）
    ├─ CRC 校验是否正确
    └─ SelTarget/SelAction 参数是否匹配芯片型号
    │
    ▼
Step 4: 环境因素
    ├─ 金属干扰（涡流效应）
    ├─ 多采集器同柜干扰
    └─ 馈线过长导致信号衰减
```

## 常见问题与解决方案

### Q1：采集器读到 EPC 但温度不刷新

**可能原因**：
1. 芯片未进入测温模式
2. SelTarget/SelAction 参数与芯片型号不匹配
3. 天线功率过低

**排查步骤**：
1. 确认芯片型号（LTU3K → SelTarget=0x07；LAU1 → SelTarget=0x06）
2. 尝试将采集器功率上调 2dBm
3. 发送 Reset 指令重置芯片状态
4. 检查是否有其他设备干扰

### Q2：读取距离明显缩短

**可能原因**：
1. 馈线过长（超过建议长度）
2. 金属反射/涡流干扰
3. 天线安装位置不佳

**排查步骤**：
1. 检查馈线型号与长度：
   - RG58: ≤10m（每米损耗 0.5dB）
   - RG316: ≤5m（每米损耗 0.9dB）
   - RG402: ≤12m（每米损耗 0.38dB）
2. 确认天线前方 0.5m 内无金属遮挡（菲涅尔区）
3. 测量 RSSI，>-65dBm 为极强，-65~-75 为良好，<-75 为临界

### Q3：多台采集器同柜干扰

**解决方案**：
1. 启用跳频模式，错开各采集器工作频段
2. 设置错时工作，避免同时发射
3. 相邻采集器使用不同天线极化方向（圆极化 vs 交叉极化）

### Q4：母线槽上传感器读不到

**可能原因**：
1. 涡流效应（传感器贴附在金属表面）
2. 传感器安装方向不对

**解决方案**：
1. 确保传感器标签垂直于铜排长边（减小涡流）
2. 优先选用抗金属型传感器（anti_metal: true）
3. 在传感器与金属之间加隔磁片

### Q5：Modbus 通信超时

**排查步骤**：
1. 确认 RS485 接线正负极
2. 检查波特率是否一致（默认 9600）
3. 测量总线电压，确认终端电阻匹配
4. 用 Modbus Poll 或 PC 端工具直接测试采集器
